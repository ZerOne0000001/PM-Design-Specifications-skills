# 需求背景文档

## 1. 需求背景 (Background)
本需求基于现有的“智慧餐饮综合管理系统”中的“移动收银”模块进行迭代升级。
目前系统已支持通过移动设备进行“按金额”支付的收银模式。
随着业务发展，食堂自助餐场景出现了新的收银需求：**按次付费**。
为了满足自助餐场景下的快速通行、防止作弊以及精细化计费（按人员类型+餐次定价）的需求，需要在现有订单系统和支付体系基础上，新增“按次付费”的业务逻辑，并适配固定的智能收银终端设备。

## 2. 目标用户 (Target Users)
*   **主要用户**：食堂就餐顾客（教职工、学生、访客等）。
*   **运营人员**：食堂管理人员（负责配置餐次规则、处理异常）。

## 3. 核心场景 (Core Scenarios)
*   **场景描述**：顾客到达食堂门口，通过固定收银设备（Android智能终端），自助完成身份识别与扣费，成功后进入食堂就餐。
*   **典型流程**：
    1.  顾客到达自助餐厅门口。
    2.  在固定设备上刷卡/扫码/刷脸。
    3.  系统根据当前时间（餐次）和用户身份（教工/学生），自动匹配价格并扣费。
    4.  设备屏幕大字显示“支付成功”，语音播报结果。
    5.  顾客入场。

## 4. 业务目标 (Business Goals)
*   **提升效率**：通过固定设备自助核销，减少人工干预，加快入场速度。
*   **灵活计费**：支持分时段（餐次）、分人群（成员类型）自动差异化定价。
*   **数据完整**：在现有订单体系中增加“业务类型”（按次/按金额）及设备信息（类型/SN码），便于后续审计与对账。
