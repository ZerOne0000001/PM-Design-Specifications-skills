# 迭代与需求日志格式规范 (Log Format)

在 `pm-workflow` 阶段 5（迭代收尾）更新追踪日志时，你**必须**严格遵守以下两种文件的表格格式。**绝对禁止**修改表头结构或增加多余的废话。

## 1. 全局迭代日志 (`docs/prd/ITERATION_LOG.md`)

该文件用于记录整个项目的版本迭代历史。
你每次更新时，**必须**在表格最下方追加一行，格式如下：

| 迭代版本 (YYMM_N) | 需求 ID 集合 | 核心迭代目标摘要 | 创建日期 (YYYY-MM-DD) |
| :--- | :--- | :--- | :--- |
| 2405_1 | F1, F2 | 初始化用户系统与登录流程 | 2024-05-10 |
| 2405_2 | F3 | 增加微信第三方支付集成 | 2024-05-15 |

## 2. 迭代内需求清单 (`docs/prd/YYMM_N/Requirement_List.md`)

该文件用于记录当前特定迭代（例如 `2405_2`）内包含的具体需求。
你每次更新时，**必须**在表格最下方追加一行，并包含对应生成的 PRD 文档链接，格式如下：

| 需求 ID | 需求名称 | 关联 PRD 链接 | 状态 |
| :--- | :--- | :--- | :--- |
| F1 | 账号密码登录 | [PRD_账号登录.md](F1_账号登录/PRD_账号登录.md) | 已完成 |
| F2 | 找回密码 | [PRD_找回密码.md](F2_找回密码/PRD_找回密码.md) | 已完成 |
