# 页面信息结构 (Page Information Architecture)

## 1. 固定收银终端 (Android)

### 1.1 登录页 (Login Page)
*   **登录方式 A**：账号 / 密码登录
*   **登录方式 B**：人脸识别登录 (点击切换)
*   **后续动作**：选择档口 (Stall Selection) -> 进入首页

### 1.2 首页 (Home Page)
*   **状态栏**：时间、网络状态、当前档口名称
*   **核心信息区**：
    *   **餐次名称** (如“午餐”)
    *   **时段** (如 11:00-13:00)
    *   **价格提示**：
        *   **基础价格** (大字显示，如 ￥15.00)
        *   **特殊价格** (小字轮播/静态提示，如“教工 ￥10.00 / 学生 ￥8.00”)
*   **操作区 (根据配置模式动态渲染)**：
    *   **状态 A (默认支付模式)**：
        *   **图示**：请刷卡/扫码 (动态图)
        *   **按钮**：【刷脸支付】(点击后弹窗/跳转识别)
    *   **状态 B (选择支付模式)**：
        *   **标题**：“请选择支付方式”
        *   **列表**：[一卡通] [微信] [支付宝] [记账] ... (网格布局)
*   **隐藏入口**：
    *   **触发**：点击左上角 Logo 5 次 / 长按空白处
    *   **动作**：弹出密码输入框 -> 验证成功进入设置页

### 1.3 密码验证页/弹窗 (Password Verify)
*   **标题**：管理员验证
*   **输入**：数字密码键盘
*   **动作**：【确认】 -> 验证后台配置的设备密码

### 1.4 本地设置页 (Settings Page)
*   **配置项**：
    *   **支付模式**：[默认支付] / [手动选择] (单选)
    *   **网络设置**：查看/配置 IP
    *   **系统信息**：版本号、设备SN
*   **动作**：【保存并返回】、【退出登录】

### 1.5 识别页 (Identify Page)
*   **入口**：状态 B 选中支付方式后 / 状态 A 点击刷脸后
*   **标题**：当前支付方式 (如“微信支付”)
*   **状态**：
    *   **监听中** (刷卡/扫码)
    *   **摄像头预览** (刷脸时)
*   **动作**：【取消/返回】

### 1.6 二次确认弹窗 (Confirmation Popup)
*   **触发**：后端返回“重复消费”状态码
*   **标题**：重复消费确认
*   **内容**：“您在 [午餐] 时段已消费 [1] 次，确认再次支付？”
*   **操作**：
    *   **【确认支付】** (高亮/大按钮)
    *   **【取消】** (灰色/小按钮)
    *   *倒计时自动取消 (10s)*

### 1.7 结果页 (Result Page)
*   **状态图标**：成功 (绿色对勾) / 失败 (红色感叹号)
*   **关键信息**：
    *   **支付金额**：￥15.00
    *   **人员类型**：教工
    *   **余额**：
        *   **默认**：`***` (脱敏)
        *   **操作**：点击“眼睛”图标显示明文
        *   **低余额提示**：若余额 < 下一餐次基础价，红字闪烁提示 *“余额不足，请充值”*
    *   **失败原因** (失败时显示)：如“余额不足”、“非营业时间”
*   **自动动作**：3秒倒计时返回首页

## 2. Web 管理后台

### 2.0 页面布局框架
*   **一级导航** (蓝底)：智慧餐厅 (激活)
*   **二级菜单** (浅底)：... | 移动收银 (选中) | ...
*   **左侧侧边栏** (白底)：
    *   订单管理
    *   退改管理
    *   **配置管理 (新增)**

### 2.1 配置管理页 (Configuration)
*   **布局**：上下分栏结构 (Vertical Layout)

#### 区块 1：基础参数设置 (Basic Parameters)
*   **标题**：基础配置
*   **表单内容**：
    *   **设备端管理员密码**：[输入框] (6位数字，用于终端本地设置验证)
*   **操作**：【保存配置】(右下角按钮)

#### 区块 2：价格策略配置 (Pricing Strategy)
*   **标题**：价格策略
*   **全局配置**：
    *   **默认/基础价格**：[输入框] (￥，所有餐次的兜底价格)
*   **列表操作**：【新增一行】(蓝色按钮，点击在表格顶部追加空行)
*   **列表交互**：**行内编辑模式 (Inline Edit)**
*   **列表字段**：
    *   **餐次** (下拉多选：[ ]早餐 [x]午餐 [ ]晚餐 ...)
    *   **人员类型** (下拉多选：[x]教工 [ ]学生 [x]访客 ...)
    *   **价格** (输入框：￥10.00)
    *   **启用状态** (开关)
    *   **操作** (保存/取消/删除)
*   **逻辑**：
    *   点击“新增一行” -> 表格顶部出现新行 -> 选择餐次/人员 -> 输入价格 -> 点击“保存”生效。
    *   已存在的行 -> 点击“编辑”进入修改模式 -> 修改后保存。
